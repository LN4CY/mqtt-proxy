## Summary

- Adds `EXTRA_MQTT_ROOTS` env var to subscribe to additional MQTT root topics beyond the node's configured root
- On MQTT connect, subscribes to `{root}/2/e/#` for each extra root, deduplicating against the node's own root
- Prevents crosstalk by filtering extra-root packets for channels the node already has
- Enables monitoring traffic from other regions when using `mqtt-proxy` with MeshMonitor's Channel Database for server-side decryption

Addresses [meshmonitor#2305](https://github.com/Yeraze/meshmonitor/issues/2305)

## Usage

```yaml
mqtt-proxy:
  environment:
    - EXTRA_MQTT_ROOTS=msh/US/OH,msh/US/CA
```
